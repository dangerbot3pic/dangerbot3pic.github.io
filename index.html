ANSI C to MIPS Compiler

The Compiler accepts C90 source code and generates the corresponding MIPS Assembly. As an added feature, it can also convert the C code into its Python equivalent.

The generated MIPS code can be assembled and linked against a C run time and then executed on a MIPS processor. I used Qemu to do this.

The Compiler uses Bison and Flex to lex and parse the input, following which the backend, written in C++ takes over. The C++ backend generates a parse tree which has distinct classes for the context and memory. This means each class is able to generate code without having to worry about the state of other classes.

Context keeps track of the scope, and variable mappings. It maintains a record of variables that have already been declared. When a new scope is entered, such as an if-else block, newly declared variables are recorded and can be used in the if-else block after which the variable will not be available to the rest of the program. The context also supports global variables and allows for local variables to be declared with the same name as the global, and any uses of that variable mean the local variable is used first. As the context keeps track of declared variables, it is also responsible for memory allocation.

A separate class keeps track of register and stack allocation. A vector of pointers to structs (that each has its own vector) is used to maintain a record of the memory/register allocation for each scope and allows for efficient use of registers.

A test framework was also created for the Compiler, with several test C sources to test general and edge cases. A bash script instructed the test case to be compiled by both my Compiler and GCC's cross compiler which was then linked using gcc and the outputs compared.

Paxos

This was a project in Distributed Algorithms which involved reading Leslie Lamport's original paper on 'Paxos Made Simple' following which I was tasked to develop my own version of the Paxos Consensus Protocol.

The project was completed in Elixir, a new language for me at the time, as it's well suited for concurrent programming.

Paxos is a family of consensus algorithms in an unreliable network, and guarantee Validity, Agreement and Liveness properties. I implemented the Basic Paxos protocol which uses two phases and distinct Clients, Acceptor and Leaders to agree on a value. A quorum of messages must be received by any proposer and only then will consensus be reached, as the the Agrement proeprty ensures that any Acceptor doesn't accept more than one value in a round.

My implementation was then used for a simulated case, wherein transactions to a banking system occured and my Paxos protocol was used to ensure that out of multiple proposed transactions, only one was selected and performed at any time.

The project took form during my first year at Imperial College London as a Group Project undertaken in my first year of univerity. The project was a showcase of the power of FGPA's in real world applications. I gained a valuable insight into High Level Synthesis and Computer Vision and Image Processing algorithms.

MIPS Simulator

In this project, I implemented the MIPS 1 ISA in a simulator written in C++. All in all, 53 instructions were implemented along with unit tests for each.

The simulator was built with an object oriented approach. One of the major tasks was having appropriate memory management, encoding using the MIPS specification and being able to handle exceptions and branches.

To ensure ease of testing, an API was provided which made all the simulators easy to use and thereby allowing each simulator to work with testbenches designed by others.

This project was tremendously useful in teaching me about memory management in C++ as well as architectural design. It solidifed my understanding of RISC architectures and was excellent preparation for another project, the ANSI C to Mips Compiler.

Spyre

This project was undertaken during a summer internship at a Cybersecurity firm. The task was to develop an End to End Encrypted Chat application.

As part of this project, I read up about key exchange protocols and implemented my own version of Diffie-Hellman Key Exchange. This approach of implementing a custom version of Key Exchange gave me the fine grained control I needed to meet the project requirements.

The biggest challenge to this project was demonstrating that the key exchange was indeed secure, this was achieved by performing attacks such as man in the middle and showing that the key exhcange was indeed secure.

Neural Networks

This is a combination of projects completed at university. Firstly, a library to implement, train and evaluate Decision Trees. And second to implement a Neural Network library and use a library for two regression and classification problems.

The Decision Tree library was written in Python with modules available for exposing the Decision Tree directly, as well as addition modules to abstract away the specifics of training, building the tree, pruning an evaluating the tree. This project was one of my personal favorites, teaching me a lot about Machine Learning and the maths behind it. In my own time, I have extended this library, implementing Random Forests.

The second part of the project, creating a Neural Networks library was, similar to the previous part, an invaluable tool in understanding the basics of the maths behind Neural Networks. Following this I used Tensorflow with the Keras wrapper for two problems. The regression problem involved predicting the position of a robotic arm based on motor input values, and the classification problem involved predicting the location of a robot based on signal strengths in different rooms. Both problems encouraged me to look into the types of error to use and to look into optimisation techniques such as using Genetic Algorithms.

NN Trading

This project is my foray into using Neural Networks to predict stock prices and use these as a trading strategy. I had spent time looking into time series analysis of signals and attempted to use these in a trading strategy. However, these are reliant on the series being stationary and hence ended up provided a very poor beta whenever a large scale upset occured, and it would take a while to once again predict prices reasonably accurately after an upset.

This brought me to Neural Networks where I planned to use not only N-Day trailing data but also qualitative methods such as sentiment analysis on news feeds etc. So far, it has proved a very promising strategy and one that has had me explore other aspects of Neural Networks such as Deep Neural Nets as well as alternative kinds of Neurons like LSTM's and CNN's.
